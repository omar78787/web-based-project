Lesson 9: References in C++

Overview

In C++, a reference is an alias for another variable. It allows you to directly refer to a variable without using pointers, making code easier to read and less error-prone. References provide the ability to pass variables to functions without copying them, much like pointers, but with a simpler and safer syntax.

Differences Between References and Pointers

  1. A reference must be initialized when it is created, while a pointer can be initialized later.

    2. int a = 10;
    int& ref = a; // Must initialize
    int* ptr = nullptr; // Can be initialized later

  2. Reassignment: A reference cannot be reassigned to refer to a different variable, while a pointer can.

    3. int a = 10, b = 20;
    int& ref = a;
    ref = b; // This does NOT change the reference to point to b. It changes the value of 'a' to 'b'.

  3. Nullability: Pointers can be null, but references must always refer to valid objects (cannot be null).

    4. int a = 10;
    int& ref = a; // ref cannot be null
    int* ptr = nullptr; // ptr can be null

Code Example:

#include
void swap(int& a, int& b) {
int temp = a;
a = b;
b = temp; }
int main() {
int x = 5, y = 10;
std::cout << "Before swap: x = " << x << ", y = " << y << std::endl;
swap(x, y);
std::cout << "After swap: x = " << x << ", y = " << y << std::endl;
return 0; }

Output:

Before swap: x = 5, y = 10
After swap: x = 10, y = 5
